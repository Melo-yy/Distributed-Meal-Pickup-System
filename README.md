# Distributed Meal Pickup System (分布式高并发取餐系统)

## 📋 项目概述
面向高校场景的分布式多端融合取餐系统，采用"端-云"异步通信架构（FlashDB本地缓存），支持高并发取餐请求。系统由主控节点（STM32F4）、从控节点（STM32F1）和 Bootloader 组成。

## 🏗 系统架构
        ┌──────────────┐
        │   Cloud/4G   │
        └──────┬───────┘
               │ 4G
        ┌──────┴───────┐
        │  主控 (F407)  │ ←── CAN Bus ──→ 多个从控（F103）
        └──────┬───────┘
               │ UART
        ┌──────┴───────┐
        │ 扫码/显示 等  │
        └──────────────┘

## 📁 项目结构
├── Code_主控 V4.0/          # 主控节点 (STM32F407)
│   ├── APP/                 # 应用层任务
│   ├── BSP/                 # 板级支持包
│   ├── FlashDB/             # 嵌入式数据库
│   ├── FreeRTOS/            # RTOS 内核
│   └── USER/                # 主函数入口
├── Code_从控 V3.0/          # 从控节点 (STM32F103)
│   ├── APP/                 # 应用层
│   ├── BSP/                 # 板级支持包
│   ├── User/                # 用户代码
│   └── Hardware/            # 外设驱动
└── Bootloader/              # 主控 IAP 引导 (F407)

## ✨ 功能特性
- **主控节点**：4G 云通信、CAN 总线组网、扫码取餐、OTA 远程升级、FlashDB 数据存储、语音播报
- **从控节点**：CAN 通信、扫码/语音播报、 显示
- **Bootloader**：IAP 固件升级引导
